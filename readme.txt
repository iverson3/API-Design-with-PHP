API接口设计
根据自己在API开发上的一点小小经验，尝试着开发的一个API接口设计方式

目录说明:
client/                     客户端调用示例
config/                     配置文件
model/                      功能模型类(逻辑处理模型类) 
model/core/                 模型基类 + 数据库连接类
controller.php              入口文件
api-normative-document.doc  API接口规范文档




注意: 

1. 上面所说的API接口规范文档，其实是我另一个项目的接口文档，但由于其格式借鉴和参考了<百度知道开放平台频道API接口规范文档>,所以还是具有很大的参考价值的
2. 测试示例中，虽然涉及到数据库，但并没有给出数据库设计，因为数据库很简单，大家可以自己去进行创建
